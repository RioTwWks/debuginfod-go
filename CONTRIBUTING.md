# Внесение вклада

Мы приветствуем ваши идеи и исправления! Пожалуйста, ознакомьтесь с правилами.

## Как предложить изменение

1. Форкните репозиторий.
2. Создайте ветку с говорящим именем (`feature/your-feature` или `fix/issue-42`).
3. Напишите код и тесты.
4. Убедитесь, что все тесты проходят (`make test`).
5. Оформите pull request с описанием ваших изменений.

## Код-стайл

- Используйте `gofmt` для форматирования.
- Комментируйте экспортируемые функции (godoc).
- Старайтесь сохранять высокое покрытие тестами.

## Лицензия

Внося изменения, вы соглашаетесь с лицензией проекта (MIT).
