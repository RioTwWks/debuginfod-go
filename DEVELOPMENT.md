# Разработка debuginfod-go

## Архитектура

Проект состоит из следующих модулей:

- `cmd/debuginfod` — точка входа, парсинг флагов, запуск сервера и индексатора.
- `internal/indexer` — сканирование файловой системы, извлечение build-id.
- `internal/storage` — абстракция над SQLite (CRUD для артефактов).
- `internal/webapi` — HTTP-обработчики для протокола debuginfod.
- `pkg/buildid` — утилиты для парсинга ELF-секций.

## Сборка

```bash
make build
```

## Тестирование

```bash
make test
```

Интеграционные тесты можно запускать с реальной базой SQLite (временный файл).

## Добавление новых функций

1. Опишите желаемое поведение в issue.
2. Напишите тесты (предпочтительно TDD).
3. Внесите изменения, следуя код-стайлу.
4. Обновите документацию.

## Профилирование

Для анализа производительности используйте `pprof`:

```bash
go test -bench=. -cpuprofile=cpu.out
go tool pprof -http=:8080 cpu.out
```

## Взаимодействие с MCP

В Cursor вы можете использовать MCP-серверы из `./.cursor/mcp.json` для ускорения рутинных задач.
