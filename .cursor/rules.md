# .cursor/rules.md
# Общие правила для разработки debuginfod сервера на Go

language: go
go_version: "1.21"

project_type: "HTTP-сервер с SQLite и парсингом ELF"

# Основные принципы
preferences:
  - Используй стандартную библиотеку там, где это разумно.
  - Для маршрутизации HTTP предпочти net/http, но можно использовать chi или gin, если упрощает код.
  - Пакет debug/elf из stdlib — основной для чтения ELF.
  - Для работы с SQLite используй github.com/mattn/go-sqlite3.

style:
  - Следуй официальному код-стайлу Go (gofmt, go vet).
  - Называй переменные и функции ясно и кратко.
  - Избегай глобальных переменных, используй dependency injection.

testing:
  - Пиши юнит-тесты с использованием стандартного пакета testing.
  - Для интеграционных тестов используй testcontainers или моки.

documentation:
  - Все экспортируемые функции должны иметь комментарии в формате godoc.
  - Основные сценарии опиши в README.md и DEVELOPMENT.md.

mcp:
  - При необходимости используй MCP-серверы для дополнительных проверок (например, для запуска линтеров или генерации документации).
